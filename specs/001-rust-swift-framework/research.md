## Research Question 1: Rust-Swift FFI Pattern

### DECISION: cbindgen with Manual C Header Oversight

**Approach**: Use cbindgen for automatic C header generation from Rust source, combined with careful manual design of the C interface layer.

### RATIONALE

For a cross-platform clipboard app requiring Rust core and Swift macOS UI:

| Factor | Evaluation |
|--------|------------|
| **Memory Safety** | cbindgen generates proper C-compatible types that work with Swift's C interop. Clear ownership semantics prevent double-free issues. |
| **Maintainability** | Automatic generation reduces manual work when Rust code changes. Explicit C interface provides clear documentation. |
| **Build Simplicity** | cbindgen integrates with Cargo build process. Standard C headers universally supported by Swift/Xcode. |
| **Performance** | Direct C ABI provides excellent performance without extra layers. Zero-copy string passing possible. |

### ALTERNATIVES CONSIDERED

- **swift-bridge**: Rejected - Overkill for simple clipboard operations, adds unnecessary abstraction, smaller ecosystem
- **Manual C Headers Only**: Rejected - Too brittle for active development, maintenance burden
- **safer-ffi**: Rejected - More complex than needed, learning curve overhead
- **objc**: Rejected - macOS-only, not suitable for cross-platform requirements

### CODE EXAMPLE

**Rust Side** (core/src/ffi/clipboard.rs):
```rust
use std::ffi::{CStr, CString};
use std::os::raw::{c_char, c_int};
use std::ptr;

#[no_mangle]
pub extern "C" fn pasty_clipboard_get_text() -> *mut c_char {
    let text = "Hello from Rust!"; // Example
    match CString::new(text) {
        Ok(c_string) => c_string.into_raw(),
        Err(_) => ptr::null_mut(),
    }
}

#[no_mangle]
pub extern "C" fn pasty_clipboard_set_text(text: *const c_char) -> c_int {
    if text.is_null() { return -1; }
    let c_str = unsafe { CStr::from_ptr(text) };
    match c_str.to_str() {
        Ok(rust_text) => {
            // TODO: Implement clipboard set
            println!("Setting: {}", rust_text);
            0 // Success
        }
        Err(_) => -1, // Error
    }
}

#[no_mangle]
pub extern "C" fn pasty_free_string(ptr: *mut c_char) {
    if !ptr.is_null() {
        unsafe { let _ = CString::from_raw(ptr); }
    }
}
```

**Swift Side** (macos/PastyApp/FFIBridge.swift):
```swift
import Foundation

class PastyFFIBridge {
    private func loadRustFunction<T>(_ name: String) -> T? {
        // Load from embedded static library
        guard let lib = dlopen(nil, RTLD_NOW),
              let sym = dlsym(lib, name) else {
            return nil
        }
        return unsafeBitCast(sym, to: T.self)
    }

    func getClipboardText() -> String? {
        typealias GetTextFunc = @convention(c) () -> UnsafeMutablePointer<CChar>
        guard let getText: GetTextFunc = loadRustFunction("pasty_clipboard_get_text") else {
            return nil
        }

        let cString = getText()
        guard let string = String(validatingUTF8: cString) else { return nil }

        // Free Rust-allocated memory
        typealias FreeFunc = @convention(c) (UnsafeMutablePointer<CChar>) -> Void
        guard let freeString: FreeFunc = loadRustFunction("pasty_free_string") else {
            return string
        }
        freeString(cString)

        return string
    }

    func setClipboardText(_ text: String) -> Bool {
        typealias SetTextFunc = @convention(c) (UnsafePointer<CChar>) -> Int32
        guard let setText: SetTextFunc = loadRustFunction("pasty_clipboard_set_text") else {
            return false
        }

        return text.withCString { cString in
            setText(cString) == 0
        }
    }
}
```

**cbindgen.toml**:
```toml
language = "C"
include_version = true
header = "/* Auto-generated by cbindgen - DO NOT MODIFY */"
autogen_warning = "/* DO NOT MODIFY THIS FILE MANUALLY */"

[fn]
prefix = "pasty_"

[enum]
prefix_with_name = true
```

**Memory Management**:
- Rust owns strings allocated with `CString::into_raw()`
- Swift must call matching `pasty_free_string()` to reclaim memory
- Use integer return codes for success/failure (0 = success, non-zero = error)
- Always check for null pointers when dealing with C strings

## Research Question 2: Build Orchestration

### DECISION: Shell Script Wrapper with Xcode Run Script Integration

**Approach**: Use shell scripts for build orchestration, integrated with Xcode via Run Script build phases.

### RATIONALE

| Requirement | Solution |
|-------------|----------|
| **Incremental Builds** | Shell scripts track modification times; Xcode handles Swift-side dependencies |
| **Debug/Release Configurations** | Scripts detect `CONFIGURATION` environment variable |
| **Universal Binary Support** | Shell scripts use `lipo` to combine x86_64 + arm64 |
| **Clear Error Messages** | Shell scripts provide verbose validation before builds |
| **CI/CD Compatible** | Pure shell scripts work in any CI environment |

### ALTERNATIVES CONSIDERED

- **Pure Xcode Build Phases**: Limited flexibility, harder to maintain across platforms
- **Swift Package Manager Plugins**: Immature ecosystem for Rust integration
- **Makefile/CMake**: Overkill for this use case, adds complexity
- **Tauri Approach**: Too specialized, not suitable for custom Swift apps

### SCRIPT STRUCTURE

**build-core.sh**: Build Rust static library
- Arguments: `[debug|release] [x86_64|arm64|universal]`
- Validates Rust toolchain version (≥1.70)
- Checks Xcode command line tools installed
- Installs Rust targets if missing (x86_64-apple-darwin, aarch64-apple-darwin)
- Builds with `cargo build --target <triple>`
- Creates universal binary with `lipo` if requested

**build-macos.sh**: Build Swift app linking to Rust library
- Arguments: `[debug|release]`
- Validates Rust library exists from build-core.sh
- Invokes `xcodebuild` with appropriate configuration
- Links against Rust static library in `target/universal/<config>/`
- Exports archive for release builds

**build.sh**: Main orchestrator
- Arguments: `[debug|release] [clean] [arch=x86_64|arm64|universal]`
- Calls build-core.sh first
- Then calls build-macos.sh
- Shows build summary with artifact locations

**Xcode Integration**: Add Run Script phase
```bash
CONFIGURATION=${CONFIGURATION:-Release}
PROJECT_DIR="${PROJECT_FILE_PATH%/*}"
"$PROJECT_DIR/../scripts/build-core.sh" "$CONFIGURATION" universal
```

## Research Question 3: DMG Packaging and Code Signing

### DECISION: create-dmg with Code Signing Workflow

**Approach**: Use create-dmg tool for professional DMG creation with proper code signing workflow.

### RATIONALE

| Tool | Verdict | Reason |
|------|---------|--------|
| **create-dmg** | ✅ RECOMMENDED | Professional layout, automation-friendly, actively maintained |
| hdiutil | ❌ Rejected | Limited layout control, manual positioning required |
| dmgbuild | ❌ Rejected | Requires Python + dependencies, slower |
| Disk Utility | ❌ Rejected | GUI-based, cannot be automated |

### SIGNING STRATEGY

**Signing Order** (critical!):
1. Sign .app bundle with `codesign --force --deep` (includes embedded Rust library)
2. Notarize signed .app (for production - requires Apple Developer account)
3. Create DMG from signed app
4. Staple notarization ticket to DMG (optional but recommended)

**Certificate Detection**:
```bash
if security find-identity -v -p codesigning | grep "Developer ID Application"; then
    SIGNING_IDENTITY=$(security find-identity -v -p codesigning | grep "Developer ID Application" | head -1 | cut -d'"' -f2)
else
    SIGNING_IDENTITY="-"  # Ad-hoc signing
fi
```

**App Bundle Signing**:
```bash
codesign --force --deep --verify --verbose \
    --sign "$SIGNING_IDENTITY" \
    --options runtime \
    --entitlements macos/PastyApp.entitlements \
    build/macos/PastyApp.app
```

**Rust Library Considerations**:
- Embedded Rust static library is included in app bundle
- Use `--deep` flag to sign all components
- No separate signing required for Rust library
- Universal binaries supported (x86_64 + arm64)

## Research Question 5: String Marshaling (Answered in Q1)

**DECISION**: C Strings with Ownership Transfer

- Rust allocates with `CString::into_raw()`
- Swift receives as `UnsafeMutablePointer<CChar>`
- Swift converts with `String(validatingUTF8:)`
- Swift calls Rust's `pasty_free_string()` to deallocate

**Key Points**:
- Always check for null pointers
- Use integer return codes for success/failure
- Document ownership transfer in FFI API

## Research Question 6: Universal Binary Support

### DECISION: Build Per-Architecture, Combine with lipo

**Build Process**:
1. Build x86_64: `cargo build --target x86_64-apple-darwin`
2. Build arm64: `cargo build --target aarch64-apple-darwin`
3. Combine: `lipo -create lib.x86_64.a lib.arm64.a -output lib.a`

**Script Integration**:
- build-core.sh accepts `arch` parameter
- Default to `universal` for production
- Supports single-architecture builds for testing

**Xcode Integration**:
- Universal Rust library linked in Xcode
- Xcode handles architecture slicing for final app bundle

## Research Question 8: Build Dependency Validation

### DECISION: check-prereqs.sh Script

**Validates**:
- Rust toolchain version ≥ 1.70
- Xcode command line tools installed
- Required Rust targets installed
- Swift compiler available
- create-dmg installed (for packaging)

**Clear Error Messages**:
```bash
if ! command -v rustc &> /dev/null; then
    echo "ERROR: Rust compiler not found"
    echo "       Install from: https://rustup.rs/"
    exit 1
fi

RUST_VERSION=$(rustc --version | cut -d' ' -f2)
if [[ "$(printf '%s\n' "1.70.0" "$RUST_VERSION" | sort -V | head -n1)" != "1.70.0" ]]; then
    echo "ERROR: Rust version $RUST_VERSION is too old (required: ≥1.70.0)"
    echo "       Update with: rustup update"
    exit 1
fi
```
