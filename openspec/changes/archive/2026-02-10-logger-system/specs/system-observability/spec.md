## ADDED Requirements

### Requirement: 全链路关键埋点
系统 **必须** 在所有关键业务路径和数据操作点添加结构化日志，以确保系统行为的可观测性。

#### Scenario: 数据库操作埋点
- **WHEN** 执行数据库操作（查询、插入、更新、删除）时
- **THEN** 系统 **必须** 记录操作类型和执行结果（INFO/DEBUG）
- **AND** 如果发生错误，**必须** 记录详细错误信息 (ERROR)

#### Scenario: OCR 服务埋点
- **WHEN** 触发、进行或完成 OCR 识别时
- **THEN** 系统 **必须** 记录请求发起、识别耗时和最终状态（成功/失败）

#### Scenario: 生命周期埋点
- **WHEN** 应用启动、退出或改变前后台状态时
- **THEN** 系统 **必须** 记录相应的状态变更事件

### Requirement: 日志等级策略
系统 **必须** 根据操作频率和重要性严格执行日志等级策略。

#### Scenario: 高频操作等级
- **WHEN** 记录高频操作（如鼠标移动、频繁的状态轮询、搜索输入过程）时
- **THEN** **必须** 使用 `DEBUG` 或 `VERBOSE` 等级
- **AND** 默认在 Release 构建中不启用输出

#### Scenario: 关键低频操作等级
- **WHEN** 记录关键的低频操作（如配置变更、应用启动、数据库迁移）时
- **THEN** **必须** 使用 `INFO` 等级

#### Scenario: 异常处理等级
- **WHEN** 系统遇到非预期错误或异常时
- **THEN** **必须** 使用 `ERROR` 等级

### Requirement: 隐私与安全
日志系统 **必须** 保护用户隐私数据。

#### Scenario: 剪贴板内容保护
- **WHEN** 记录剪贴板相关的日志时
- **THEN** 系统 **必须** 记录条目的 ID、类型、来源应用
- **BUT** 系统 **必须禁止** 记录剪贴板的文本内容、图片数据或文件路径等敏感信息
