# Dangerous Action Confirmation

## Purpose
为危险操作提供二次确认机制，防止用户误操作。

## ADDED Requirements

### Requirement: 恢复默认设置需要用户确认
系统必须在用户点击"恢复默认设置"按钮时显示确认对话框。

#### Scenario: 用户确认恢复默认设置
- **WHEN** 用户点击"Restore"按钮
- **THEN** 系统显示确认对话框，标题为"确认恢复默认设置？"
- **AND** 对话框包含说明文字"此操作将重置所有设置为默认值，且无法撤销。"
- **AND** 对话框提供"取消"和"恢复"两个按钮
- **WHEN** 用户点击"恢复"按钮
- **THEN** 系统将所有设置重置为默认值
- **AND** 系统保存更新后的设置
- **AND** 系统关闭确认对话框

#### Scenario: 用户取消恢复默认设置
- **WHEN** 用户点击"Restore"按钮
- **THEN** 系统显示确认对话框
- **WHEN** 用户点击"取消"按钮
- **THEN** 系统关闭确认对话框
- **AND** 设置保持不变

### Requirement: 清空历史记录需要用户确认
系统必须在用户点击"清空历史记录"按钮时显示确认对话框。

#### Scenario: 用户确认清空历史记录
- **WHEN** 用户点击"Clear All"按钮
- **THEN** 系统显示确认对话框，标题为"确认清空历史记录？"
- **AND** 对话框包含说明文字"此操作将永久删除所有剪贴板历史，且无法撤销。"
- **AND** 对话框提供"取消"和"清空"两个按钮
- **WHEN** 用户点击"清空"按钮
- **THEN** 系统删除所有剪贴板历史记录
- **AND** 系统关闭确认对话框

#### Scenario: 用户取消清空历史记录
- **WHEN** 用户点击"Clear All"按钮
- **THEN** 系统显示确认对话框
- **WHEN** 用户点击"取消"按钮
- **THEN** 系统关闭确认对话框
- **AND** 历史记录保持不变

### Requirement: 确认对话框使用警示样式
系统必须使用危险操作（destructive）样式标识确认操作，以警示用户。

#### Scenario: 恢复默认设置对话框样式
- **WHEN** 系统显示"恢复默认设置"确认对话框
- **THEN** "恢复"按钮使用危险操作样式（红色）

#### Scenario: 清空历史记录对话框样式
- **WHEN** 系统显示"清空历史记录"确认对话框
- **THEN** "清空"按钮使用危险操作样式（红色）
